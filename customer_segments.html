Implementation: Outlier Detection¶

Detecting outliers in the data is extremely important in the data preprocessing step of any analysis. The presence of outliers can often skew results which take into consideration these data points. There are many "rules of thumb" for what constitutes an outlier in a dataset. Here, we will use Tukey's Method for identfying outliers: An outlier step is calculated as 1.5 times the interquartile range (IQR). A data point with a feature that is beyond an outlier step outside of the IQR for that feature is considered abnormal.

In the code block below, you will need to implement the following:

  • Assign the value of the 25th percentile for the given feature to Q1. Use np.percentile for this.
  • Assign the value of the 75th percentile for the given feature to Q3. Again, use np.percentile.
  • Assign the calculation of an outlier step for the given feature to step.
  • Optionally remove data points from the dataset by adding indices to the outliers list.

NOTE: If you choose to remove any outliers, ensure that the sample data does not contain any of these points!
Once you have performed this implementation, the dataset will be stored in the variable good_data.

    Q1 = np.percentile(log_data[feature], 25)

    Q3 = np.percentile(log_data[feature], 75)

    step = 1.5 * (Q3 - Q1)

  • Good performance with large datasets compared to hierarchical clustering.
  • The algorithm guarantees to converge at a local minimum.
  • Although, the algorithm stops at a local minimum, it is possible to set the number of iterations before hand.

Disadvantages:

  • Very sensitive to starting points which can lead to bad clustering.
  • Works poorly when the dataset has non-convex shapes or it has different densities of points.
  • Since it's based on means, the presence of outliers has a big influence when defining the clusters.

Gaussian Mixture Model

Unlike K-Means, GMM is a soft clustering method based on the likelihood of a point to belong to a cluster.

Advantages:

  • It's more flexible when it comes to assign points to a cluster. Since it's based on the likelihood, an observation may share characteristics of several sub-populations (clusters). It is possible to set a threshold to define a point as part of a cluster.
  • The influence of outliers can be minimized.
  • It is very fast for learning mixture models.
  • It improves the likelihood at every iteration.

Disadvantages:

  • Similar to K-Means, it's very sensitive to the initial conditions.
  • It does not converge to a local minimum. It improves the likelihood at every iteration, but it does not find the best configuration (in practice, it does).
  • It can get stuck, that means that it can reach a point with no good likelihood, but it's not able to improve.

For the currrent problem, I will choose GMM for the following reasons:

  • I did not remove any outlier, so K-Means may fail.
  • It behaves better for datasets with different densities. This is something that may happen in the current dataset given the scattered graphs of the features.

Question 10¶

Companies often run A/B tests when making small changes to their products or services. If the wholesale distributor wanted to change its delivery service from 5 days a week to 3 days a week, how would you use the structure of the data to help them decide on a group of customers to test?
Hint: Would such a change in the delivery service affect all customers equally? How could the distributor identify who it affects the most?

Answer:

I would recommend to test changes in a subset of the more compact cluster, the one which observations are closer to each other. In our current example, that is the cluster 1. Those observations tell me that those customers tend to have a more similar behaviour when acquiring the products. So, when testing the changes, it's likely they would have a similar reaction to them, which would let the wholesale distributor give a proper response.

When selecting the customers to test, I would recommend to start from the center of the cluster and define a radius of selection. The observations within that radius would be chosen ones for the testing.

I would recommend an iterative process for testing, in every iteration, the radius of the selection area would increase a given amount (we can use variance to define how much). The selected elements for the new iteration would be those within the radius except the ones that were already tested. Then, it is possible to compare the reactions and affectations with the ones of the previous iteration, therefore, identifying which are the customers that are more affected by the changes.

Another approach for testing is to select a cluster, then subdivide it in smaller clusters and apply what was mentioned above. This leads to an iterative process that can stop by defining a minimum number of observations per cluster.
